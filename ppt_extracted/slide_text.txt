LIVENESS DETECTION — ANTI-SPOOFING SYSTEM 6 of 12 MULTI-LAYER LIVENESS VERIFICATION PIPELINE 1 👁️ BLINK DETECTION Eye Aspect Ratio (EAR) algorithm on landmark points. EAR < 0.25 = blink. Requires 1–2 genuine blinks. 2 ↔️ HEAD TURN DETECTION Nose-to-eye distance ratio changes across frames. Requires left & right turn. >15° deviation detected. 3 😊 SMILE DETECTION Lip corner landmark ratio measurement. Mouth width vs height ratio triggers smile verification. 4 🔬 TEXTURE ANALYSIS Silent passive check. CNN detects moiré/ print artifacts of photos and screens (< 5ms). 🎲  RANDOMIZED CHALLENGE SEQUENCE Challenges are randomly selected and sequenced each session to prevent replay attacks. Minimum 2 of 3 active challenges must pass. 🚫 Photo Attack Texture CNN + EAR flatline detection catches printed photos 📺 Screen Replay Moiré pattern analysis + reflection artifacts detected passively 🎭 3D Mask Depth cue inconsistency + lighting gradient analysis ⏱ Timeout Max 10 sec per challenge — auto-fail on timeout HACKATHON 7.0  |  Datalake 3.0 Offline Biometric Authentication

SYNC & PURGE MECHANISM 8 of 12 01 🔐 Authentication Event Field auth completes offline. Result encrypted & stored in SQLite pending queue. → 02 📶 Network Monitor NetInfo listener polls continuously. Triggers on any available connection. → 03 ☁️ Batch Upload All pending records batched. HTTPS POST to API Gateway with JWT authentication. → 04 ✅ Server Validation Lambda validates payload, stores in DynamoDB, returns acknowledgement token. 05 SECURE DATA PURGE (on acknowledgement) After receiving server ACK token, SQLite pending queue records are cryptographically wiped (zero-fill overwrite). Face embeddings never stored permanently on device. Audit log of sync event retained only (no biometric data). 🔑 AES-256 Encryption  —  All local records encrypted at rest using device-derived key from SecureStore 🔒 TLS 1.3 in Transit  —  HTTPS-only sync — no plaintext transmission of any biometric data 📵 Offline-First Design  —  System functions identically with or without connectivity — sync is additive only 🗑 Guaranteed Purge  —  Server ACK required before purge — prevents data loss on failed upload HACKATHON 7.0  |  Datalake 3.0 Offline Biometric Authentication
