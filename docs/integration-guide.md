# Datalake 3.0 Integration Guide

## Install

Copy the `src/faceguard` folder into the Datalake 3.0 React Native app or publish it as the internal package `@datalake/faceguard`.

Required runtime dependencies:

```bash
npm install react-native-vision-camera expo-secure-store expo-sqlite @react-native-community/netinfo uuid react-native-get-random-values
```

For bare React Native, use equivalent native modules for secure storage, SQLite, and TFLite. Keep the public FaceGuard API stable.

## App Setup

Wrap the app with `FaceGuardProvider`:

```tsx
import { FaceGuardProvider } from './src/faceguard/FaceGuardProvider';

export function Root() {
  return (
    <FaceGuardProvider>
      <AppNavigator />
    </FaceGuardProvider>
  );
}
```

## Authentication Screen

Use the hook from any screen:

```tsx
import { useFaceAuth } from './src/hooks/useFaceAuth';
import { FaceGuard } from './src/components/FaceGuard';

export function FaceGuardScreen() {
  const { authenticate, status, lastError, lastResult } = useFaceAuth();

  return (
    <FaceGuard
      status={status}
      onVerify={authenticate}
      error={lastError?.reason}
      result={lastResult}
    />
  );
}
```

## Real Camera Integration

Replace `createDemoFrameSequence()` in `useFaceAuth.ts` with frames generated by `react-native-vision-camera`. The rest of the pipeline stays unchanged.

Expected frame fields:

- `timestamp`
- normalized luminance
- texture score or raw frame input for the anti-spoof adapter
- detected face box and landmarks, if already produced by native code

## Real TFLite Integration

Implement `ModelAdapter`:

```ts
export type ModelAdapter = {
  initialize(): Promise<void>;
  detectFace(frame: FrameSample): Promise<FaceDetection | undefined>;
  estimateTextureLiveness(frame: FrameSample): Promise<number>;
  createEmbedding(frame: FrameSample, face: FaceDetection): Promise<number[]>;
};
```

Then inject it into `FaceGuardEngine`.

The current Android build intentionally avoids the incompatible Nitro-based
`react-native-fast-tflite` v3 runtime because Expo SDK 51 uses React Native
0.74. Keep the working local enrollment/authentication path active, then add
trained TFLite inference after choosing a runtime verified against the exact
React Native version.

## Sync Endpoint

Set the API endpoint in `AwsSyncClient`:

```ts
new SyncService(queue, new AwsSyncClient('https://your-api-gateway/faceguard/sync'));
```

The endpoint must return:

```json
{
  "acknowledgedIds": ["queue-id-1"],
  "serverAckToken": "signed-server-token"
}
```

Rows are purged only after their `queueId` appears in `acknowledgedIds`.
